# Интернет-магазин GymBoss | Full-stack учебный проект

**Стек:** PHP, MySQL, JavaScript, HTML/CSS, Docker, Nginx.  
**Статус:** Основной функционал реализован. Проект в активной разработке.

## Описание проекта

GymBoss — учебный интернет‑магазин спортивных товаров, реализованный с нуля как **full‑stack** веб‑приложение.  
Проект используется для прокачки навыков backend, frontend, работы с БД, Docker, веб‑серверами, внешними API и фреймворками.

## Функционал

- **Frontend** — адаптивный интерфейс на чистом JS:
    - Регистрация и авторизация пользователей
    - Личный кабинет
    - Оформление, оплата, просмотр и отмена заказов
    - Редактирование и удаление аккаунта
    - Работа с формами
- **Backend:** PHP с регистрацией, авторизацией пользователей, взаимодействием с базой данных MySQL.
- **База данных:** MySQL. Спроектированы и реализованы таблицы для товаров, заказов, пользователей.
- **Интеграции с внешними API:**
    - **Платежи:** ЮKassa (приём онлайн‑оплат).
    - **Доставка:** Яндекс Карты (выбор пункта выдачи или адреса доставки).
    - **Валидация:** DaData (подсказки адресов).
    - **Уведомления:** SMSC (отправка SMS с кодом подтверждения телефона).
- **Безопасность и лучшие практики:**  
  В проект постепенно внедряются рекомендации по безопасной веб‑разработке:
    - Строгая валидация входных данных на backend и ограничения полей ввода на frontend
    - Использование корректных HTTP‑статусов, try/catch, транзакций для критичных операций и логирования с контекстом
    - CSP, экранирование HTML/JS при выводе, защита от XSS/CSRF, безопасные cookie‑файлы
    - Хэширование паролей пользователей
- **SEO и сервисные страницы:**
    - Страницы политики конфиденциальности, контактов и магазинов
    - Динамический robots.txt
    - Скрипт генерации актуальной sitemap.xml

## Установка и запуск

Для установки нужны git и Docker.

Шаги установки:

1. Клонировать репозиторий:  
   `git clone https://github.com/mocus8/gym-boss-website.git`  
   `cd gym-boss-website`
2. Отредактировать файл окружения `.env.example` (тестовые ключи можно получить на соответствующих сайтах).
3. Создать и запустить Docker‑контейнеры:  
   `docker compose up -d --build`

После этого сайт будет локально доступен по указанному в `.env` адресу.
